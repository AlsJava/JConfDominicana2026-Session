# Resource

Esta carpeta contiene guías prácticas para la charla — análisis profundos sobre el pensamiento arquitectónico, las decisiones de ingeniería y las prácticas de desarrollo que dan forma a la base de código. Cada archivo cubre un aspecto específico de cómo trabajamos, tomamos decisiones y estructuramos la solución.

---

## Contenido

| Archivo | Qué aprenderás |
|---------|----------------|
| [DEVELOPMENT-es.md](DEVELOPMENT-es.md) | Paradigmas de arquitectura (Hexagonal, Microservicios, Tradicional/MVC), cómo el tamaño del equipo impulsa las elecciones arquitectónicas, marcos de decisión y el patrón Command + CQRS usado en este proyecto |
| [AI-es.md](AI-es.md) | Cómo usar la IA como herramienta — no como reemplazo — para el desarrollo; configurar AGENTS.md, prácticas de supervisión y casos de uso de alto valor como mensajes de commit, descripciones de PR y reportes de dependencias |
| [MAINTAINABILITY-es.md](MAINTAINABILITY-es.md) | Los 5 niveles de documentación (arquitectura → código), por qué la documentación como código importa, cómo diseñar documentación que funcione tanto para humanos como para herramientas de IA |
| [SOLUTION_PERFORMANCE-es.md](SOLUTION_PERFORMANCE-es.md) | Rendimiento más allá de Big-O: índices de base de datos, protocolos de serialización (JSON, Protobuf, Kryo), Jackson Blackbird y extracción de consultas vía Spring Boot AOT |
